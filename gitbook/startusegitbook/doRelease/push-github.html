发布到Github仓库

发布到Github仓库

GitBook.com 还可以集成GitHub，所以用户可以将书籍的源码通过 GitHub 上的项目来管理，这样可以使用 GitHub 带来的各种优点，例如：

  • 其它用户可以 fork
  • 用户可以点赞，获得更新提醒
  • 用户可以贡献自己的内容

创建 GitHub 项目

首先，登陆 GitHub，然后创建一个 test 项目，如下图所示：

导入代码

现在，将书籍原来的 Gitbook 项目内容导入到新建的 GitHub 中的项目中，进入Gitbook项目

$ cd test

// 
$ git remote add github https://github.com/USER_NAME/test.git

$ git push -u github master

git push命令中的-u表示将本地 master 分支的上游分支设置为 github/master，所以以后修改了本地 master 分支后，git push将推送到 github 上。

使用 GitHub 上的项目

进入gitBook，对应项目的setting 找到GitHubIntegration，进行设置

保存后，可以看到之前不可点击的 "Add a deployment webhook" 按钮已经可以点击了，这个按钮表示：每当用户配置的 GitHub 上的项目更新时，自动更新书籍！
